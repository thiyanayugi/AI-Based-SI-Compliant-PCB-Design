# Contributing to AI-Based SI-Compliant PCB Design

Thank you for your interest in contributing to this project! This document provides guidelines for contributing.

## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker to report bugs
- Describe the issue in detail with steps to reproduce
- Include relevant notebook outputs or error messages

### Suggesting Enhancements

- Open an issue to discuss new features or improvements
- Provide clear use cases and expected benefits
- Consider backward compatibility

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes with clear, descriptive messages
6. Push to your branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Development Guidelines

### Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and modular

### Notebook Guidelines

- Clear markdown explanations between code cells
- Include visualization of results
- Document model architecture and hyperparameters
- Show training metrics and evaluation results

### Testing

- Test notebooks end-to-end before submitting
- Verify model training converges
- Check prediction accuracy meets standards
- Test with different input configurations

## Commit Message Guidelines

Use conventional commit format:

- `feat:` New features
- `fix:` Bug fixes
- `docs:` Documentation changes
- `refactor:` Code refactoring
- `test:` Adding or updating tests
- `chore:` Maintenance tasks

## Questions?

Feel free to open an issue for any questions about contributing.
